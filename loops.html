"for" loops

How many times does the following code print "0"?
for i in range(8): print("0")

(a) 7

(b) 8

(c) 10

(d) 9

Correct!

Recall, the range function uses a start, stop and step. By default the start is 0 and the step is 1, so how many times does range(8) run?

Nested "for" loops

How many times does the following code print "0"?
for i in range(10): for j in range(10): print("0")

(a) 100

(b) 10

(c) 11

(d) 110

Correct!

Notice that the second for function is inside of the first for function. This means that it runs every time the first one runs.

Nested "for" loops

How many times does the following code print "Hello!"?
for i in range(6): for j in range(8): print("Hello!")

(a) 42

(b) 48

(c) 62

(d) 50

Correct!

Consider how the inner for loop runs. If it prints a certain amount of times, and that happens based on the outer loop, how many times total does it print?

Dynamic nested "for" loops

What is the value of x at the end of the function?
x = 0 for i in range(4): for j in range(i): x += 1

(a) 5

(b) 3

(c) 4

(d) 6

Correct!

Notice that the second for loop runs based on what the iterator i of the first for loop is.

Dynamic nested "for" loops

What is the value of y at the end of the function?

(a) 21

(b) 15

(c) 6

(d) 17

Correct!

This one is very similar to the last question. Notice that the first iterator is incremented two more times. How does this affect the second?

ASCII "for" loops

What does the following print?
my_string = "" for i in range(5): my_string += "-" print(my_string)

(a) "-"

(b) 5 "-", each on a new line

(c) A singular line of 5 "-"

(d) It causes an error

Correct!

This also tests some knowledge of string operations. Recall that "+=" adds content to the end of a string.
